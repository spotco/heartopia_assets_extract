# Heartopia audio extraction summary

## Steps used to get the audio

1. Inventory the game's Wwise audio assets with `python .\scripts\inventory_assets.py` to confirm the bank and WEM layout under `xdt_Data\StreamingAssets\Audio\GeneratedSoundBanks\Windows`.
2. Extract embedded WEM files from music-related `.bnk` files with `python .\scripts\extract_embedded_wems.py`, because much of the useful audio is stored inside bank `DIDX`/`DATA` chunks instead of only as loose `.wem` files.
3. Build naming candidates with `python .\scripts\name_unnamed_music_from_banks.py` by mapping media IDs back to bank names and source paths.
4. Analyze the candidate audio with `python .\scripts\analyze_unnamed_audio.py` so the report includes duration and basic music-vs-ambience classification.
5. Improve generic labels so bank-only names like `Mus_map` and `Amb_general` become stable unique labels such as `map_music_<media_id>` and `general_ambience_<media_id>`.
6. Clear `extracted\music_mp3` and export the playable files with `python .\scripts\export_named_music.py --overwrite`, using `vgmstream` plus `ffmpeg` to convert the decoded WEM data into MP3s.
7. Final result: **677** MP3 files exported successfully. **4** tiny `Music_metronome` blobs could not be exported because `vgmstream` could not open those 24-53 byte sources.
